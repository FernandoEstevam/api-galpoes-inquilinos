# Tipo de commit:
# feat:     Uma nova funcionalidade
# fix:      Correção de bug
# docs:     Apenas mudanças na documentação
# style:    Mudanças de formatação (semântica, indentação...)
# refactor: Refatoração sem mudança funcional
# perf:     Melhorias de performance
# test:     Adição/modificação de testes
# chore:    Alterações de build, configs, dependências, etc.

<tipo>: <descrição breve da mudança>

# Exemplo:
# feat: adiciona endpoint POST /contracts com validação Zod

# Corpo da mensagem (opcional):
#
# - O que foi feito?
# - Por que foi feito?
# - Possui breaking change?

# Rodapé (opcional):
#
# BREAKING CHANGE: descreva a mudança incompatível
# Closes: #123 (issue relacionada)
